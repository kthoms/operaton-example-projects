# Loan Platform SSO

A secured, single-sign-on loan-origination platform: the loan-application
process runs on a webapp-less Operaton engine behind Keycloak, operated through
Flowset Control and Tasklist, all behind one nginx/oauth2-proxy TLS gateway.

## What this project demonstrates

- Operaton **Run** with built-in webapps disabled, exposing only `/engine-rest`
- Keycloak as the identity provider via the Operaton Keycloak plugin (`operaton-keycloak-all`)
- Single sign-on across the welcome page and Flowset apps with nginx + oauth2-proxy
- External-task service work done by a standalone, non-Spring Java worker
- A themed welcome page and Keycloak login skin
- Redoc-rendered REST API docs at `/engine-rest`

## Architecture overview

The stack runs entirely in Docker Compose as 11 containers on a single bridge network.
A single external port — **8080** on the host — maps to nginx's TLS listener.
All browser traffic passes through nginx (TLS termination) → oauth2-proxy (OIDC
session check) before reaching any application service.

Container roles:

| Layer | Containers |
|---|---|
| Entry point / TLS | `nginx` |
| SSO gateway | `oauth2-proxy` |
| Identity provider | `keycloak`, `keycloak-init` (one-shot seeder) |
| Process engine | `operaton` (REST-only, Keycloak plugin) |
| Service worker | `worker` (external-task, plain Java) |
| Task management | `flowset-control`, `flowset-tasklist` |
| Databases | `postgres-operaton`, `postgres-keycloak`, `postgres-flowset` |

The engine uses `ENGINE_AUTH_MODE=BASIC_IDP` (credentials validated by
`KeycloakIdentityProviderPlugin` against Keycloak ROPC). Browser-facing SSO is
handled independently by oauth2-proxy. See `docs/arc42` for the full views.

## Process model

![Loan application](engine/resources/loan-application.png)

The `loan-application.bpmn` process:
1. **Submit loan application** (start form: `loanAmount`)
2. **Score credit** — external task topic `credit-scoring` (worker derives `creditScore`)
3. **Risk assessment** — DMN decision table `risk-assessment.dmn` (evaluates `creditScore` → `riskLevel`)
4. **Gateway** — low/high risk routes auto-complete; medium risk routes to a user task
5. **Underwriter Review** — user task (candidate group: `underwriter`); medium-risk path only
6. **Low/High paths:** worker handles topic `notification`, sets `loanDecision=APPROVED` or
   `REJECTED`; **Medium path:** process waits at the Underwriter Review user task — no
   notification external task on this path.
7. End events: `EndEvent_Approved` or `EndEvent_Rejected`

## Prerequisites

- **Docker** (recent, with Compose V2)
- **JDK 21** — only required to run the automated integration test
- A self-signed TLS certificate in `nginx/certs/` — generated by `./nginx/gen-cert.sh`
- (Optional, for rendering the process diagram) `npm install -g bpmn-to-image`

Credentials (all dev/throwaway values):

| User | Password | Keycloak group | Role |
|---|---|---|---|
| `alice` | `alice` | `employees` | Can start loans and use Tasklist |
| `eve` | `eve` | `underwriter` | Completes Underwriter Review tasks |
| `admin` | `admin` | `operaton-admin` | Full engine access |
| `worker` | `worker` | `operaton-admin` | Service account for the worker |

## Run it

```bash
cd examples/loan-platform-sso

# 1. Generate the self-signed TLS certificate (once)
./nginx/gen-cert.sh

# 2. Start all services
docker compose up -d --build

# 3. Open the application (accept the self-signed certificate warning)
# https://localhost:8080/
```

The stack is ready when `docker compose ps` shows all containers healthy.
Startup takes approximately 60–90 seconds on first run (Keycloak initialisation).

## Walk through it

1. **Open** https://localhost:8080/ — nginx redirects to the Keycloak login.
2. **Log in** as `alice` / `alice` (or `eve` / `eve`). After authentication, the
   welcome page appears with tiles for Control and Tasklist.
3. **Start a loan** — click **Tasklist**, then **Start Process** → **loan-application**.
   Enter a `loanAmount` (e.g. `100000`) and submit.
4. The **worker** picks up the `credit-scoring` task: a loan amount ≤ 500 000
   produces `creditScore = 720` (low risk). The DMN table maps this to
   `riskLevel = low`. The worker then handles the `notification` task and sets
   `loanDecision = APPROVED`. The process completes automatically.
5. For a **medium-risk** path: inject `creditScore = 650` as a start variable.
   The process reaches the **Underwriter Review** user task (group `underwriter`).
   Log out, log in as `eve`, open Tasklist, claim the task, and complete it.
   The process ends directly — `loanDecision` is set by an execution listener on the
   outgoing sequence flow; there is no separate notification external task on this path.
6. For a **high-risk** path: `loanAmount > 500000` or inject `creditScore = 500`.
   The process routes directly to `REJECTED` without a user task.
7. **Monitor** all instances in **Control** at https://localhost:8080/control/.
8. **Browse the REST API** at https://localhost:8080/engine-rest — Redoc renders
   the Operaton OpenAPI specification inline.

## How it works

- **Engine image** — `engine/Dockerfile` builds from `operaton/operaton:2.1.1`
  (Run distribution), downloads `operaton-keycloak-all-2.1.0.jar` into
  `userlib/`, and replaces the default config with `engine/config/default.yml`.
  It starts with `--rest` (no Cockpit/Tasklist/Admin WARs).
- **Authentication** — see `docs/arc42/09-architecture-decisions.md` (ADR-001).
  `BASIC_IDP` mode: HTTP Basic credentials on `/engine-rest` are validated by
  `KeycloakIdentityProviderPlugin` against Keycloak's ROPC endpoint.
- **Worker** — `src/main/java/.../LoanWorker.java` and `KeycloakTokenProvider.java`.
  A plain Java `main` class, no Spring. Handles topics `credit-scoring` and
  `notification` using the Operaton external-task client.
- **Gateway** — `nginx/nginx.conf` uses `auth_request /oauth2/auth` for every
  protected route. `oauth2-proxy/oauth2-proxy.cfg` configures the Keycloak OIDC
  provider with PKCE and cookie-based sessions.
- **Keycloak seeding** — `keycloak/seed-realm.sh` runs once in the
  `keycloak-init` container and creates the `operaton` realm, clients, groups,
  and users idempotently via `kcadm.sh`.
- **Welcome page** — `nginx/html/index.html` (itemis-branded) served directly
  by nginx after the oauth2-proxy auth check passes.
- **Runtime and deployment diagrams** — `docs/arc42/06-runtime-view.md` and
  `docs/arc42/07-deployment-view.md`.

## Run the tests

```bash
# Maven (runs integration tests via failsafe)
./mvnw verify

# Gradle
./gradlew build
```

`LoanPlatformIT` uses Testcontainers to start Postgres, Keycloak, and the custom
engine image, seeds the realm, and drives the loan-application process through
the in-process worker for three scenarios: **low risk** (auto-approved),
**high risk** (auto-rejected), and **medium risk** (waits for the Underwriter
Review user task). No `Thread.sleep` — Awaitility polls the engine history API.
